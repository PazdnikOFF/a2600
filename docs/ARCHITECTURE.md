# Эндоскопическая стойка — архитектура ПО (реконструкция X-2000)

> Рабочий документ. Цель — построить собственное ПО, функционально эквивалентное
> референсной стойке **X-2000**, для идентичного железа (Zynq UltraScale+ EV).
> Не копирование бинарников, а **реимплементация** на том же открытом стеке,
> со сверкой поведения по живому эталону. Стадия применения: R&D / стенд.

---

## 1. Цели и границы

| | |
|---|---|
| **Что делаем** | Своё приложение осмотра: живое видео эндоскопа → снимок/запись → архив → отчёт → DICOM/печать |
| **Что переиспользуем как есть** | `libhal.so`, `system.bit` (битстрим PL), firmware модулей, базовый rootfs PetaLinux, штатные конфиги `system/*` |
| **Что пишем заново** | Прикладной слой (`X2000` + `X2000Video`), опираясь на карту классов референса |
| **Не в скоупе (пока)** | Клиническая валидация, регуляторика, применение на пациентах |

**Принцип реконструкции:** из референса извлекаются *имена классов, методов и типы аргументов* (Itanium mangling) → восстанавливается **API-поверхность/заголовки**. Логика функций и приватные структуры недоступны (DWARF пустой) → поведение **реимплементируется** и сверяется с эталоном (железо идентично, можно гонять рядом).

## 2. Целевая платформа

- **SoC:** Xilinx **Zynq UltraScale+ MPSoC, вариант EV** — Cortex-A53 ×4 (aarch64) + Mali-400 (GPU) + **VCU** (аппаратный H.264/H.265).
- **ОС:** PetaLinux / Yocto. Тулчейн `aarch64-xilinx-linux` (GCC 8.2), sysroot `/opt/sdk/sysroots/aarch64-xilinx-linux`.
- **Дисплей:** DRM/KMS `fd4a0000.zynqmp-display` (DisplayPort/DPDMA), мультиплановый композитинг.
- **Видеозахват:** PL-фабрика (битстрим) → Xilinx Video Pipeline → V4L2 `/dev/video0` (NV12). Настройка топологии — `media-ctl`.
- **Кодек:** OMX над VCU (`omxh264enc` / `omxh264dec`).
- **GPU:** Mali-400, Qt eglfs (EGL/GLES2).

Опознано по строкам бинарников: `fd4a0000.zynqmp-display`, `a00a0000.v_tpg`, `omxh264enc`, `/opt/sdk/sysroots/aarch64-xilinx-linux`, `libMali.so.9`.

### Идентификация референса

Референс — **SonoScape X-2600** (OEM SonoScape), «Sistema de Videoendoscopia Integrado». Ребренды: **«РуСкейн»** (РФ), ST Endoscopia (Испания). Класс — интегрированная стойка видеоэндоскопии, совместимая с **гибкими, жёсткими и фиброэндоскопами**.

Спецификации из брошюры (влияют на архитектуру):
- **Видеовыходы:** 1080p, **2×DVI + 2×SDI** → многомониторность.
- **Панель управления:** сенсорный экран **8″** (front panel) — отдельный HMI.
- **Источник света:** 2-LED, ≥50 000 ч, авто/ручной, яркость **уровни 1–19**.
- **Цифровой зум:** до **4×**. Баланс белого: авто + запоминаемый. **AGC** (усиление).
- **Хранение:** запись фото/видео на USB-память и внешний HDD.
- **Автоопределение типа эндоскопа** → автоматическая генерация соответствующего интерфейса.
- Питание 100–240В, 350ВА, 16.7 кг.

### UI-топология (мультидисплей)

```
                       ┌───────────────────────────┐
 основной монитор ◄────┤ Zynq US+ : zynqmp-display  │   видео-плоскость (kmssink)
 (1080p, DVI/SDI)      │  + Mali (Qt eglfs)         │ + UI-плоскость (Qt OSD, alpha)
                       └───────────────┬────────────┘
                                       │  (KLcdProxy / видеовыход)
                       ┌───────────────┴────────────┐
 8″ сенсорная панель ◄─┤ HMI (отд. firmware)        │   кнопки/параметры (скрин 1)
 управления            │  MCU: класс HmiMcu         │   ← hmi_upd_*.bin
                       └────────────────────────────┘
```

- **Основной монитор** (скрин 2): эндоскопическое видео + OSD-статус (IRIS, Zoom, Img/Col Enh, Col RBC, AGC, Light Mode, Contrast) + карточка пациента (Name/Gender/Age).
- **8″ сенсорная панель** (скрин 1): оперативное управление — отдельный HMI с собственной прошивкой (`hmi_upd_*.bin`, класс `HmiMcu`, обновляется секцией `[hmi]`). Ещё есть модуль `[lcd]` (`lcd_upd_*.bin`) и `[pap]/[papp*]` — прошивки суб-контроллеров/обработки.

## 3. Контекст системы: процессы и IPC

```
        ┌──────────── базовая прошивка устройства (НЕ в пакете обновления) ────────────┐
        │  capture-служба: FPGA/USB (Hal_UsbFpga) или V4L2 → кадры в разделяемую память │
        └───────────────────────────────┬──────────────────────────────────────────────┘
                                         │ SysV shm (kadr NV12)
        ┌────────────────────────────────┴───────────────────────────────┐
        ▼                                                                  ▼
┌──────────────────────────┐     KProcShMemory / KProcShmManager    ┌──────────────────────┐
│  X2000  (главный, Qt5)    │◄──────── shared memory ───────────────►│  X2000Video (рендер) │
│  • UI, логика осмотра     │                                        │  • GStreamer + Mali  │
│  • KVideoProxy (IPC)      │  ──── команды/события (shm/сокет) ────► │  • kmssink → экран   │
│  • DICOM, архив, отчёты   │                                        │  • видео-плоскость   │
│  • GStreamer снимок/запись│                                        └──────────────────────┘
│  • /dev/mem: рег. FPGA    │
└───────────┬──────────────┘
            │ dlopen/link
            ▼
      libhal.so  (749 Hal_* C-API: питание, свет, зонды, ЭКГ, навигация, принтер…)

      X2000Monitor   — вотчдог/супервизор процессов
      X2000Simulator — эмулятор железа для разработки без прибора
```

**Ключевые факты IPC (из статики):**
- Кадры передаются через **SysV shared memory** (`shmget/shmat/shmdt`), менеджер — `KProcShMemory` / `KProcShmManager`, реестр сегментов `_KProcShm`.
- Композитинг: **видео на отдельной DRM-плоскости** (kmssink), **UI Qt на графической плоскости**, смешивание по alpha (`modetest -D fd4a0000.zynqmp-display -w 36:alpha:N`).
- Часть инициализации FPGA делает сам X2000 через `/dev/mem` (регистры `0xA00600xx`) + маркер `/dev/fpgainit.ini`.
- Смена формата видео — скрипт `system/platform/switchvideoformat.sh` (media-ctl + v4l2-ctl).

## 4. Видеотракт (полностью раскрыт)

### 4.1 Захват
```bash
# 1) топология PL-пайплайна (пример 1280×960)
xmedia-ctl-nv12 -d /dev/media0 -V '"a00a0000.v_tpg":0 [fmt:VYYUYY8_1X24/1280x960 field:none]'
# 2) формат V4L2-узла
v4l2-ctl -d /dev/video0 --set-fmt-video width=1280,height=960,pixelformat=NV12
```
Захват — **стандартный V4L2** (`/dev/video0`, **NV12**). Для базового видео проприетарный API не нужен.

### 4.2 Рендер (production)
`v4l2src device=/dev/video0 io-mode=dmabuf ! video/x-raw,format=NV12 ! kmssink bus-id=fd4a0000.zynqmp-display`
Zero-copy через dmabuf, вывод на видео-плоскость DRM. UI Qt — сверху, по alpha.

### 4.3 Снимок / запись (из строк референса)
- Пайплайны: `kImagePipeline` (снимок), `kRecoderPipeline`/`KEncoderPipeline` (запись).
- Кодирование записи — `omxh264enc` (VCU) → mux → файл.
- Снимок — кадр из тракта → OpenCV/Qt → JPEG/PNG и/или DICOM Secondary Capture.

### 4.4 Обработка / экспозиция
- `K3ADimming` — авто-яркость/экспозиция (3A) под конкретный сенсор.
- Сенсоры: **IMX274** (осн., по типам эндоскопа), OV6946, OV2740, OAH0428, OH01A.
- Гамма-профили: `system/videoconf/Gamma/<SENSOR>_GammaParam_<SCOPE>.ini`.
- Управление источником света: `Hal_Dccu*`, класс `KDccuParam`.

### 4.5 Раскладки экрана
`system/display/IMG<WxH>-UI<WxH>.ini` — прямоугольники областей (видео, PIP, списки, статус) для комбинаций «размер кадра × разрешение UI» (UI 1920×1200 / 1920×1080).

### 4.6 Режимы освещения и цветовые режимы (chromoendoscopy)

Проприетарные режимы SonoScape (аналоги NBI) — ключевая функциональная ценность:

| Режим | Расшифровка | Суть |
|---|---|---|
| **WL** | White Light | Обычный белый свет |
| **EWL** | Enhanced White Light | Усиленный белый свет — мелкие сосуды видны чётче |
| **SFI** | Spectral Focused Imaging | Спектральное усиление изображения (оптика + цифра) |
| **VIST** | Variable Intelligent Staining Technology | «Виртуальное окрашивание» слизистой/сосудов |

SFI/VIST = комбинированная оптико-цифровая обработка (подсветка 2-LED + цифровые матрицы). В реконструкции — реализуются как цифровые LUT/матрицы преобразования над кадром (кандидат: OpenCV/шейдер на Mali) + управление источником света. Точные коэффициенты — из `system/videoconf/*` и `coldlight/*`, либо подбор по эталону.

### 4.7 Параметры изображения (карта OSD/панели)

| Параметр | Значения | Ориентир (референс) |
|---|---|---|
| **ENH** (Img Enh) | 1 / 2 / 3 (OSD: A0…) | резкость/детализация |
| **COLOR** (Col Enh) | 1 / 2 / 3 | цветовое усиление |
| **Col RBC** | +15 / 0 / −15 | усиление красного (сосуды/кровь) |
| **IRIS** | PEAK / AVE / AUTO | режим экспозамера → `K3ADimming` |
| **ZOOM** | 1 / 2 / 3, до **4×** | цифровой зум |
| **AGC** | On/Off | авто-усиление яркости |
| **Contrast** | L / M / H | контраст |
| **Image Brightness** | **1–19** (Auto/Manu) | яркость света → `Hal_Dccu*` |
| **LEVEL** | L / M / H + **AIR** | уровень (подача воздуха/инсуффляция) |
| **WHT BAL** | — | баланс белого (авто + запоминаемый) |

### 4.8 Автоопределение эндоскопа
Система сама определяет тип подключённого эндоскопа и строит соответствующий UI/пресеты (классы `KEndoScope`, `KScopeInfoEdit`, `KCameraInfoEdit`). Серия эндоскопов **X20**: гастроскоп (EG-X20), колоноскоп (EC-X20/L), бронхоскоп (EB-X20/T), ринолариноскоп (ENL), холедохоскоп (ECH), цистоскоп (ECY), уретероскоп (EUC) + камерные головки F18 для жёстких/фибро и лапароскопы.

## 5. HAL — переиспользование `libhal.so`

749 функций `Hal_*` (C-API, `extern "C"`). Приложение зовёт их напрямую (линк `NEEDED libhal.so`). Основные группы:

| Группа | ф-ций | Назначение |
|---|---|---|
| `Hal_Get*` / `Hal_Set*` | 178 | Чтение/запись параметров |
| `Hal_PowerBoard*` | 59 | Плата питания/управления |
| `Hal_UsbFpga*` | 58 | Интерфейс к FPGA по USB (конфиг/прошивка/поток) |
| `Hal_PimUsb*`/`PimApp*` | 51 | Probe Interface Module (зонды/датчики) |
| `Hal_System*` | 34 | Системные операции |
| `Hal_Read*`/`Write*` | 40 | Низкоуровневый I/O |
| `Hal_ATC3DG*` | 18 | Электромагнитная 3D-навигация |
| `Hal_EcgModule*` | 15 | ЭКГ-синхронизация |
| `Hal_Iot/Net/WiFi/Beacon` | 26 | Связь/IoT |
| `Hal_VideoDisp*` | 6 | Управление выводом видео |
| `Hal_Dccu*` | 3 | Источник холодного света (диммирование) |
| прочее | FingerPrint, Dvd, TouchPad, KeyBoard, Gpio, Prober… | периферия |

Стратегия: обернуть нужные `Hal_*` тонким C++-фасадом (`hal/Hal.h`), железо не трогать.

## 7. Данные и хранение

- **БД:** SQLCipher (`libsqlcipher`) — пациенты, осмотры, снимки, настройки (зашифрованная SQLite).
- **DICOM:** DCMTK — экспорт снимков/видео как **Secondary Capture**, отправка в PACS, worklist. (Пересекается с проектом PACS Интер-Мед.)
- **Изображения:** OpenCV 4.3 + libtiff/webp/jpeg.
- **Конфиги:** ini-файлы `system/*` (display, videoconf, coldlight, printer, presetdata, language, style) — переиспользуются как есть.
- **Пути на устройстве:** приложение — `/home/root/`, данные — `/home/root/data/`, конфиги — `/home/root/system/`, съёмный носитель — `/tmp/media/`.

## 9. Сборка

- **Кросс-компиляция** через PetaLinux/Vitis SDK (`aarch64-xilinx-linux`), `source environment-setup-*`.
- **Sysroot** — с устройства/образа (Qt5, GStreamer с Xilinx-плагинами omx/v4l2, Mali, DCMTK, OpenCV, libhal).
- **CMake** + toolchain-файл (`cmake/toolchain-aarch64-xilinx.cmake`).
- **Qt QPA:** `QT_QPA_PLATFORM=eglfs` (Mali EGL/GLES2).
- Нативная сборка на приборе возможна, но кросс — предпочтительнее.

## 10. Дорожная карта

| Веха | Содержание | Зависимости |
|---|---|---|
| **M0** | «hello video»: `gst v4l2src → kmssink` на приборе | штатный V4L2 |
| **M1** | Скелет Qt5-приложения: окно + живое видео (appsink→виджет для bring-up) | Qt5, GStreamer |
| **M2** | Production-рендер: kmssink на видео-плоскости + UI-оверлей по alpha | DRM/KMS |
| **M3** | Снимок + запись (VCU, `omxh264enc`), сохранение файлов | M1 |
| **M4** | Архив снимков + БД (SQLCipher) + привязка к пациенту | M3 |
| **M5** | Обработка изображения (OpenCV): ENH/COLOR/RBC/Contrast/зум + 3A-диммирование (libhal) | M1, HAL |
| **M6** | Режимы SFI/VIST/EWL (LUT/матрицы + управление светом) | M5 |
| **M7** | Панель управления 8″ (HMI): экран скрина 1, связь с осн. приложением | M1 |
| **M8** | DICOM Secondary Capture + PACS | M4, DCMTK |
| **M9** | Отчёты, измерения, настройки, аккаунты, автоопределение эндоскопа | M4 |

**Приоритет пользователя:** живое видео (M0–M2) → снимок/запись/архив (M3–M4) → обработка (M5).
**Двухмониторность (M7):** основной монитор (видео+OSD) и 8″ сенсорная панель — раздельные UI; на этапе M1–M6 фокус на основном мониторе.

## 11. Открытые вопросы / риски

1. **Кто в базовой прошивке кладёт кадры в shm** — уточнится через `strace` на приборе. Обход: брать кадры напрямую из V4L2, не завися от чужого shm.
2. **Точные версии Qt5/GStreamer/OpenCV** — снять из sysroot платы.
3. **Формат сегмента shm** (`_KProcShm`, размеры, заголовок кадра) — восстановить, если понадобится совместимость с `X2000Video`; иначе не нужен.
4. **`system.bit` и версия PL** должны соответствовать твоей плате (железо идентично — ок).
5. **Лицензии:** переиспользуются только штатные системные библиотеки прибора и открытый стек; проприетарные бинарники референса не распространяются.
