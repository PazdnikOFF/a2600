# Панель управления 8″ (HMI) — реверс раскладки

> Скрин 1 (сенсорная панель управления). В оригинале — **отдельная прошивка
> Cortex-M** (bare-metal), не часть Qt/Zynq-приложения. Этот документ — результат
> шага 1–2 методологии (реверс + поиск файлов). Код панели (шаг 3) — после решения
> по подходу (отдельная прошивка МК vs 2-й Qt-экран на Zynq).

## 1. Микроконтроллер (сигнатура, требует подтверждения по чипу)

- Ядро: **ARM Cortex-M** (Thumb-2; строки фолт-хендлера «Usage fault … ARM»).
- Карта памяти: SRAM с базой **`0x10000000`** (нач. SP `0x10007948`), код/приложение с **`0x1a040000`** (вероятно загрузчик в `0x1a000000` + app по +`0x40000`).
- Вектор-таблица: ~**46 IRQ**; слово-маркер `0x5A5A5A5A` в позиции vec[7].
- ФС: **FAT12** (диск `F:/`), библиотека GUI своя (рисует из `F:/*.bin`).
- Прошивка: `history/hmi_upd_56_21_00_00_01_17.bin` (~230 КБ, raw). Апдейтер `cmd/hmiupdate` (aarch64) шлёт по сокету.
- Локализация строк — испанский («Siguiente», «Vista»).

**Как подтвердить модель МК:** прочитать маркировку чипа на плате панели, либо сверить необычную базу flash `0x1a040000` и число IRQ с даташитами кандидатов.

## 2. Ресурсы иконок — готовая FAT12-ФС (диск F:/)

`history/lcd_upd_56_23_00_00_00_05.bin` — **образ FAT12 на 8 МБ** (boot `MSWIN4.1`, метка `LCD_UPD`, `55AA`). Это и есть `F:/`. Монтируется как обычный FAT.

**Формат иконки `.bin`:** заголовок 4 байта (`05 xx xx xx`), далее пиксели **BGR888** (3 байта/пиксель), объём = `W·H·3 + 4`. Подтверждено декодированием (FROZEN0 = 48×48, снежинка на циановом фоне).

Размеры (из размера файла, BGR888):

|---|---|---|---|
| 6916 | 2304 | **48×48** | frozen0/1, save0/1, see0/1, setup0/1, trans0/1, usb0/1, moire0/1, left, right, video1 |
| 19012 | 6336 | **88×72** | airon, airoff, bala0/1 |
| 15604 | 5200 | **80×65** | contr1/2/3 |
| 4324 | 1440 | **40×36** | auto0/1, manual0/1 |

| 550 / 592 / 196 | — | мелкие | lamp, logo1-6 |

## 3. Карта панели (кнопка → иконка(и) → состояние)

Каждая кнопка — пара `0`/`1` (нормальная/выбранная — cyan-подсветка на скрине 1).

**Верхняя строка:** `lock`/`unlock` (замок), статус STORAGE/SCOPE/SPARE, `usb0/1`, `setup0/1` (шестерёнка), `logo*`, `date0/1`/`time0/1`.

**Нижний ряд действий:**
| Кнопка (скрин) | Иконка F:/ |
|---|---|
| View | `see0/1` / `look` |
| FREEZE | `frozen0/1` |
| SNAP | `save0/1` |
| RECORD | `video0/1` |
| SET | `setup0/1` / `sys` |
| TRANS | `trans0/1` |
| EJECT | `fork0/1` |

**Правый блок:**
| Раздел | Иконки |
|---|---|
| Light Mode: WL/SFI/VIST | текст + подсветка |
| WHT BAL | `bala0/1` |
| LIGHT (лампа) | `lamp`, `on`/`off` |
| Brightness −/+ | `minus0/1`, `plus0/1` |
| AUTO / MANU | `auto0/1`, `manual0/1` |
| LEVEL L/M/H | `contr1/2/3` |
| AIR | `airon`/`airoff` |

**Левый блок параметров** (ENH, COLOR, IRIS, ZOOM) — сегментированные кнопки со значениями `1/2/3`, `PEAK/AVE/AUTO` — текстовые (не иконки).

## 4. Точные координаты

Наземный эталон координат — **скрин 1** (пиксельные позиции). Таблицы координат в прошивке требуют дизассемблера raw Thumb-2 без символов (сложно) — при необходимости добираются отдельно. Разрешение панели — уточнить (8″ TFT, кандидаты 1024×600 / 800×600; крупный фон `onch/onen` ~163200 px).

2. Иконки берём из готовой FAT12 (`F:/`) как есть (BGR888) — не конвертируем в отдельные PNG.
